控件:标题控件



标准控件的大多数版本被设定成两种形式-一个基础版本和一个会输出 控件名字的标签 的对应版本.例如, Textfield 和 TitleText.

带标题的版本实际是围绕里面包含的控件做的封装器,而不是另外方式做了适配的控件,这会是一个在修改它们的动作的时候会造成混乱的原因.[直接粗暴修改基础版的控件,带标题的版本也会受基础版影响,产生不必要的动作和属性继承]

通常,创建你自己的这些控件的新版本的时候,你应该先创建内部要包含的控件,然后在创建一个带标题的版本.

举个例子:
class NewTextWidget(textbox.Textfield):
        # all of the custom code for this class
        # should go here.


class TitleProductSearch(TitleText):
        _entry_type = NewTextWidget

你可以通过传递 begin_entry_at参数到构造函数来调整子控件[输入框]在屏幕上被放置的位置.其默认值是16.你同样可以重写控件,决定是否给标题一个独立的行,这个可以通过在控件创建时传递 use_two_lines=True|False 参数来做到.默认的 use_two_lines=None 设置会把标题和包含的控件放在同一行,除非标签是在太长的情况除外.

你可以在创建的时候使用 labelColor='LABEL'参数来改变标签的颜色.你也可以使用你当前正在使用的主题中的任何颜色.

创建完成后,由 TitleWidget对象 管理的两个小控件可以通过 label_widget 和 entry_widget 属性来访问.

带标题的多行控件



如果你正在创建带标题版本的多行控件的话,你会发现从 TitleMultiLine类继承反而会更好,因其功能上封装了更多的多行控件功能.
